== Scope and Standard

Evaluates the `verification-ecosystem/misinformation-defence-platform`
repository against *Component Readiness Grades v2.0 (STRICT)*,
2026-03-30 revision. The repo is an *umbrella* containing three
subdirectory projects (`disinfo-nesy-detector/`, `algorithm-shield/`,
`defensive-multiplicity/`) and is self-declared *C* in a flat
`[crg] grade = "C"` block under `.machine_readable/6a2/STATE.a2ml`
(last-updated 2026-04-04). The audit checks the claim against on-disk
evidence and the v2.0 "honest D > dishonest B" clause.

- Standard: `standards/component-readiness-grades/COMPONENT-READINESS-GRADES.md`
- Criteria boilerplate: `rsr-template-repo/docs/governance/CRG-CRITERIA.adoc`
- Audit template: `rsr-template-repo/docs/governance/CRG-AUDIT-TEMPLATE.adoc`
- Self-declared grade (prior art):
  * `.machine_readable/6a2/STATE.a2ml` → `[crg] grade = "C"`,
    `achieved = "2026-04-04"`, `test-count = 57`.
  * `TEST-NEEDS.md` header: "CRG Grade: C — ACHIEVED 2026-04-04".
  * `EXPLAINME.adoc` 2026-04-04 rewrite presents three subprojects as
    substantially implemented.
  * `README.adoc` (older, 2026-02-24) still states *"Project Status:
    Specification Phase — This repository currently contains project
    infrastructure only"* — the README and STATE directly contradict.
  * `TOPOLOGY.md` (2026-02-19) puts all three platform components at
    *10%* completion.
- Audit date: 2026-04-18
- Auditor: Jonathan D.A. Jewell

Schema note: umbrella STATE.a2ml uses a *sectioned* `[crg] / grade = "C"`
block, not the `[metadata] crg-grade = "C"` flat key. HYP-S005 pattern
matching the flat `[metadata]` form will not fire by default; a
sectioned-block extension of HYP-S005 would. See §HYP-S005 canary.

The audit grades the repo *as-is today*, not aspirationally.

=== Scope — "misinformation defence" vs on-disk reality

The repo name promises a platform for *detection, analysis, and
response* to misinformation. On-disk today:

- *Detection*: `disinfo-nesy-detector/eval/` contains working
  *baseline* classifiers (keyword match, TF-IDF, majority). These are
  textbook baselines; they detect at the level of a CS-intro
  assignment. The detector *service* (the neuro-symbolic NATS-ONNX-
  Souffle pipeline described in README/EXPLAINME/TOPOLOGY) is 165
  LOC of stub across four files. ONNX runtime is disabled pending
  model availability (STATE `[blockers]`). Souffle integration is
  placeholder. Dgraph integration is interface-only. The neuro-
  symbolic claim is *architectural aspiration*, not today.

- *Analysis*: not a separate component. `eval/` does evaluation, not
  analysis-of-real-content.

- *Response*: `defensive-multiplicity/` is supposed to be this. On
  disk it has 62 files, one of which is a `.res` source
  (`examples/SafeDOMExample.res`). No `src/`, no `Cargo.toml`, no
  domain code. Justfile says every verb is "not configured yet".
  This component is at *pre-E*.

- *Algorithm-shield*: the most-implemented subproject — a browser
  extension with ReScript + Rust WASM + minikaren rule engine.
  Substantive code. But its scope is "confuse filter bubbles", which
  is *adjacent* to misinformation defence (it changes what you see),
  not *defence* (it doesn't detect or respond to disinfo).

*Scope-vs-implementation verdict*: the repo implements *at most one
sub-component of one of its three stated pillars* (the baseline
classifier evaluation harness, which is not even the service path of
the detector pillar). "Misinformation defence platform" is a broad
claim that the current code base does not begin to substantiate.
This is the heaviest finding in the audit.
