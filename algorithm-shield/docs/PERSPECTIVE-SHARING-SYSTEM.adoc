= Perspective Sharing: Walk a Mile in Someone Else's Algorithmic Shoes
:toc:
:toc-placement!:

A lightweight system for importing/exporting algorithmic experiences to build empathy and understanding.

toc::[]

== The Vision

**"What does YouTube look like for a Black woman? A rural conservative? A teenager in Mumbai?"**

Instead of *hypothetical* personas, users can share their **actual algorithmic experience** and others can temporarily "try it on."

**Goal**: Build empathy by experiencing the internet from others' perspectives

== Core Concept: Algorithmic Experience Profiles

An "Algorithmic Experience Profile" (AEP) is a lightweight data structure that captures:

1. **What the algorithm shows you** (feed composition)
2. **How the algorithm treats you** (recommendations, search results)
3. **What you never see** (filter bubble boundaries)

**NOT captured** (privacy):
- Actual browsing history
- Personal information
- Login credentials
- Anything identifiable

**Captured** (anonymized):
- Feed category distribution (e.g., "70% Gaming, 20% Memes, 10% Commentary")
- Recommendation patterns (e.g., "Algorithm suggests more extreme content over time")
- Bubble boundaries (e.g., "Never see: Politics, News, Education")
- Diversity metrics (e.g., "Simpson's Diversity Index: 0.32")

When user clicks "Try This Perspective":

**Safe Mode (Default)**:
1. Opens 4-5 tabs that represent this perspective's feed
2. **Does NOT engage** (no clicks, no watch time, no likes)
3. User can visually see the difference
4. Close tabs immediately - no profile contamination

**Immersive Mode (Advanced)**:
1. Temporarily modifies your feed to match this profile
2. Runs for 15 minutes (time-boxed)
3. Records what algorithm shows you
4. Reverts to your profile afterwards
5. Generates comparison report

## Empathy-Building Use Cases

**1. Gender Bias**

```
Your perspective (Male):
  Search "CEO" → Results: 80% male CEOs, suits, offices
  Recommendations: Business, entrepreneurship, finance

"Woman in tech" perspective:
  Search "CEO" → Results: 60% male CEOs, 40% "women in leadership"
  Recommendations: "Work-life balance", "Imposter syndrome", "Women in tech"

🎓 Insight: Algorithm assumes women need different content,
even for same search query. It's profiling, not personalizing.
```

**2. Racial Bias**

```
Your perspective (White):
  Search "professional hair" → Results: Generic office styles
  Recommendations: Grooming tips, barbershops

"Black woman" perspective:
  Search "professional hair" → Results: "Natural hair in workplace"
  Recommended: "Natural hair discrimination", "Protective styles"

🎓 Insight: Algorithm assumes different "professional" standards
by race. It's reinforcing stereotypes.
```

**3. Age Bias**

```
Your perspective (35 years old):
  Feed: News, tutorials, long-form content
  Recommendations: Depth, analysis, educational

"Teenager" perspective:
  Feed: Memes, gaming, trends, shorts
  Recommendations: Viral content, quick hits, challenges

🎓 Insight: Algorithm creates age-segregated realities.
Teens and adults see completely different internets.
```

**4. Political Bias**

```
Your perspective (Centrist):
  Feed: Mix of perspectives, diverse sources
  Recommendations: News from various outlets

"Rural conservative" perspective:
  Feed: 70% conservative sources, 30% centrist
  Recommendations: More extreme versions of same viewpoint

🎓 Insight: Algorithm amplifies existing leanings,
creates polarization through recommendation gradient.
```

**5. Accessibility**

```
Your perspective (Able-bodied):
  Feed: Videos with/without captions, mixed quality
  Accessibility: Not a priority in recommendations

"Deaf user" perspective:
  Feed: Higher rate of captioned content (algorithm learns)
  BUT: Caption quality poor, many videos still inaccessible

🎓 Insight: Algorithm adapts to disability, but platforms
don't enforce caption quality. Adaptation ≠ accommodation.
```

== Privacy & Ethics

**Strict Privacy Rules**:

1. **No PII**: Profiles must strip all personally identifiable information
2. **Opt-in only**: Users must explicitly consent to sharing
3. **Anonymity levels**: Users choose how much to reveal
4. **Right to delete**: Profiles can be revoked from repository
5. **No tracking**: Repository doesn't track who downloads what

**Ethical Consent**:

```json
{
  "ethical_consent": {
    "consents_to_sharing": true,
    "understands_public": true,
    "age_verified": true,
    "can_revoke": true,
    "anonymity_level": "high",
    "signature": "a3f2b1c4...",
    "timestamp": "2026-01-24T12:00:00Z"
  }
}
```

**Anonymization Process**:

```rescript
module ProfileAnonymizer = {
  let anonymizeProfile = (rawFeedState: Observer.FeedState.t): algorithmicProfile => {
    // STRIP personal data
    let anonymized = {
      feedComposition: calculateCategoryDistribution(rawFeedState),
      bubbleBoundaries: identifyBubbleBoundaries(rawFeedState),
      diversityScore: calculateDiversityMetrics(rawFeedState),
      // NO watch history, NO search queries, NO identifiers
    }

    // ADD noise for differential privacy (if requested)
    if anonymityLevel == High {
      addLaplaceNoise(anonymized, epsilon: 0.1)
    } else {
      anonymized
    }
  }
}
```

== Technical Implementation

**File Format**: JSON (human-readable, lightweight)
**File Extension**: `.aep` (Algorithmic Experience Profile)
**File Size**: 5-10KB (tiny!)

**Import/Export API**:

```rescript
module ProfileSharing = {
  // Export current experience as profile
  let exportProfile = (
    feedState: Observer.FeedState.t,
    metadata: {
      description: string,
      anonymityLevel: anonymityLevel,
    }
  ): string => {
    let profile = {
      version: "1.0",
      id: generateUUID(),
      created: Date.now(),
      platform: "youtube",
      metadata: metadata,
      feedComposition: extractFeedComposition(feedState),
      bubbleBoundaries: extractBubbleBoundaries(feedState),
      // ...
    }

    JSON.stringify(profile, null, 2)  // Pretty-printed JSON
  }

  // Import profile from JSON
  let importProfile = (jsonString: string): result<algorithmicProfile, string> => {
    try {
      let parsed = JSON.parse(jsonString)

      // Validate schema
      if !validateProfileSchema(parsed) {
        Error("Invalid profile format")
      } else {
        Ok(parseProfile(parsed))
      }
    } catch {
    | _ => Error("Failed to parse profile")
    }
  }

  // Apply profile (Safe Mode: view only)
  let applyProfileSafeMode = (
    profile: algorithmicProfile
  ): array<string> => {
    // Generate URLs that would match this profile's feed
    let sampleUrls = generateRepresentativeUrls(profile.feedComposition)

    // Open tabs (view only, no engagement)
    sampleUrls->Array.map(url => {
      ChromeTabs.Tabs.createWithUrl(url, ~active=false)
    })

    sampleUrls  // Return for logging
  }
}
```

== Example Workflow

**Scenario: Understanding racial bias in search**

1. **User A (Black woman)** exports her profile
   - Notices search results assume race (hair care, fashion)
   - Exports anonymized profile with description
   - Shares file via Discord server

2. **User B (White man)** imports the profile
   - Downloads .aep file
   - Clicks "Try Perspective" in extension
   - Opens 4 tabs with User A's typical feed

3. **User B sees**:
   - Same search query → completely different results
   - Algorithm makes racial assumptions
   - "Professional hair" → "Natural hair in workplace"

4. **User B gains empathy**:
   - "I never realized the algorithm does this!"
   - Shares insight on social media
   - Downloads 3 more perspectives to learn more

**Result**: Algorithmic bias becomes **experiential**, not just conceptual
